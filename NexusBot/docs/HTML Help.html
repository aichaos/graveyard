    Adding/Removing Bots
  • Back to Main
     
    Your CKS NexusBot program should have come with two sample bots
    in the "bots" folder.

    To Add Bots
    To add a bot, simply create a new text document in the "bots" folder
    containing the information as shown below.

    To Remove Bots
    To remove a bot, just delete one of the text documents in the "bots"
    folder, or move it to another location.

    Standard AIM Bot Template
    The text file for an AIM bot should be formatted similarly to this:
    ScreenName: bot's screenname
    Password: bot's password
    Client: AIM
    Profile: ./data/aim/profiles/profile.html
    Buddies: ./data/aim/buddies/buddylist.txt
    Font: ./data/aim/fonts/font styles.txt

    Standard MSN Bot Template
    An MSN bot's file should be formatted similarly to this:
    ScreenName: bots_email_address@domain.tld
    Password: bot's password
    Client: MSN
    Nick: default MSN nickname
    Font: ./data/msn/fonts/font styles.txt

    Additional Variable Fields
    The additional variables as shown in the standard bots above
    (the "Profile, Buddies, and Font" for the AIM bot, and the "Nick"
    and "Font" in the MSN bot) are for additional ways to customize
    each bot.

    For more information about the standard AIM and MSN information,
    click here.

    You can add as many variables as you want to each of these
    bots. You can then call this information like this:
    $chaos->{'bot's screenname'}->{'variable name'}

    Creating User Permissions
  • Back to Main
     
    About the Permissions
    Clients
     The Client status is reserved for most normal users who talk to
     your bot. The Client status allows the user access to all public
     bot commands.
    Gifted
     The Gifted status is reserved for users who the botmaster grants
     special permissions to. The Gifted users have access to some
     commands that the normal Clients do not have access to.
    Keeper
     The Keeper status is reserved for the botmaster to assign
     permissions to other bots to keep track of their bot. The Keeper
     status typically has controls that keep the bot online and healthy.
    Moderator
     Moderators are the first level of higher permission level. Moderators
     typically have the ability to add or remove users from the blocked
     lists or warners lists.
    Super Moderator
     Super Moderators typically have the power to use the MSN Alerts
     command, as well as command the bots to set an away status or
     return from an away status.
    Administrator
     Admin users are the third top permission level in the bot. Admin
     users have commands such as listing or joining open MSN
     conversations.
    Super Administrator
     Super Admins are the second-in-command controllers of your bot.
     Super Admins have more control over the MSN conversations than
     Admin users do. Super Admins have the ability to close an MSN
     conversation, for example.
    Master
     The Master status is reserved for the botmaster alone. The Master
     status has the highest powers and should only be for the botmaster.
     In the wrong hands, somebody with Master permissions could manipulate
     anything on the bot's host computer. So it is highly recommended that
     you do not grant Master priveleges to ANY user other than yourself.

    Creating the Master User
    To create the Master User, you must sign your bot on to at least one Instant
    Messenger and send at least one non-command message to the bot. This will
    create your initial Client Data. You can then edit your Client Data (in the "clients"
    folder) and change the "Permission" field to read "Permission=Master".

    Promoting Other Users
    Once the Master User has been created, you can then promote other users to
    any status level you want. However, as a security precaution, the Master can
    NOT promote ANY other user to the Master status.

    Configuring the Webserver
  • Back to Main
     
    About the Webserver
    The CKS NexusBot webserver is an add-on to the NexusBot code that
    allows you and your bot's users to talk to the bot using the Internet.

    Requirements
    The CKS NexusBot webserver would require the following:
    A fast connection to the Internet (cable or higher)
    A static IP address. If your IP changes every time you connect, it's not good for servers

    Set Up The Server
    All you need to do to setup the server is open the file "data/server.txt" in a text
    editor such as Notepad and change the 4-digit number to any four numbers you
    want. This is the server's port. If you want to make it your default webserver, set
    the port number to 80.

    Access the Server Locally
    To access the server locally (on your own computer), go to http://localhost:port/.
    The default port is 2001. So if you do not change the port number, the address is
    http://localhost:2001/

    Access the Server Remotely
    To access the server remotely (from a different computer), you will need to know your
    computer's IP address. If you do not know your computer's IP address, you can use the
    link below to find out your IP address:
  • http://ip.chaos.kirsle.net/

    Additional Bot Information
  • Back to Main
     
    Here is some information that your bots may use.

    AIM Bot Information
    Profile
     This is the location to the webpage your bot will use as its
     AIM profile. The webpage has to be local, you cannot use
     an HTTP address for this variable. The default place to keep
     your local profiles is at "./data/aim/profiles".
    Buddies
     This is the bot's buddylist. The buddylist is just a text file
     with one buddy on each line. The buddylist is formatted as
     follows:
     group name=buddy's screenname
    Font
     This is the bot's font settings. The font settings are also just
     a plain text document in the "./data/aim/fonts" folder. The
     font file is formatted as follows (the blue fields are editable):
     Font Name=Verdana
     Font Color=black
     Background=white
     Link Color=blue
     Font Size=2
     Style=<i>
     The "Style" field is the font style. This uses simple HTML markup.
     Some examples are: <i> for Italics, <b> for bold, and <u> for underline.
    MSN Bot Information
    Nick
     The bot's default nickname (display name).
    Font
     The bot's font settings. These are similar to the AIM bot's font
     settings, except that the font's color has to be in reverse HEX
     format. An example is as follows:
     Font Name=Verdana
     Font Color=FF0000
     For more information about the MSN fonts, click here.

    Teal          999900

    Tutorial // Creating a Command
  • Back to Main
     
    This is a quick tutorial that will tell you the basics of creating a NexusBot command
    of your own.

    Variables Sent to All Commands
    The variables that are sent to every command are as follows:
     $self     - The current messenger's object that can do
                      messenger-specific actions
     $client   - The current user's handle or screenname.
     $msg      - The current user's message.
     $listener - The messenger the client is speaking from.

    Variable Returned from Command
    The variable that is returned from the command is always the reply the bot gives for
    the command. You should always use the "return" function at the end of the
    command's code.

    Creating your First Command
    In this tutorial, we're going to show you how to create a very simple "Guess My
    Number" game. The bot already came with a Guess My Number game that uses
    the command "!guess", so we're going to name our command "!gmn"

    1. Open a text editor such as Notepad and save a file in the "commands/client"
    folder. Name it "gmn.pl"
    2. At the top of the file, write "sub gmn {" This will create a handler for the command.
    3. On the line right after the one you wrote in Step 2, insert this code:
     my ($self,$client,$msg,$listener) = @_;
    This will get all the variables being sent into your command.
    4. After the text in Step 3, write the following code:
     # Pick a random number.
     my $number = int(rand(10)) + 1;
     This will make $number be a value between 1 and 10.
     if ($msg eq $number) {
     If the user's message is our number...
      $reply = "Yes! You got my number! :-D";
      Tell them they got the right number.
     }
     else {
     If they didn't get the right number...
      $reply = "Nope! It was $number! :-P";
      Tell them what the number was.
     }

     return $reply;
     Return the reply so it can be sent to the person talking.
    5. End the sub using this code:
     }
     1;
    The bracket ends the sub. The 1; is required for all sub-files the bot uses when it
    starts up (yes, a command is a sub-file). What the 1; does is it returns "true" to the
    script loading the file. That's so the script knows that the file was loaded successfully.
    6. Save the file, restart your bot, and type !gmn 5 or any number you want and start playing!

    When you are finished with this tutorial, your command should look like this:
    sub gmn {
     my ($self,$client,$msg,$listener) = @_;

     # Pick a random number.
     my $number = int(rand(10)) + 1;

     if ($msg eq $number) {
      $reply = "Yes! You got my number! :-D";
     }
     else {
      $reply = "Nope! It was $number! :-P";
     }

     return $reply;
    }
    1;
